Description

This is a 'Robin Y Hood' proxy for Youtube. If you set it as your HTTP proxy, then the Youtube videos will have an option to cache the data. For more information, please refer to the title page

Credits

There are a few software tools are used for this proxy:
  • youtube-dl The Python program, which is able to retrieve video data from Youtube
  • Go programming language fast and nice programming language
  • go proxy proxy server, written on the Golang
  • Gorilla Golang web toolkit. The mux and websocket pachages are used
  • Testify mainly for assertion library
  • go-ini to read the *.ini config files
  • groupcache for the implementation of the LRU-cache
  • go-logging as a log library
Frontend uses libraries:
  • Video.js a video player to make use of saved video files
  • jQuery no need to introduce this one
  • Pure css framework
  • Go programming language color scheme from this site

Known bugs

  • Youtube search doesn't stop the video, only hides it. If it was playing, you will hear it sound until another video is opened.
  • Every video starts from beginning. The start-from-time URL argument is not supported.
  • Service-style (not proxy-style) caching will add 'select your language' dialog to all pages
  • Some videos can not be downloaded. The ones that are not supported by youtube_dl, and the ones without *.mp4 version
  • Video is only played via flash player; only *.mp4 videos are supported
  • Video player is replaced, and all annotations and subtitles are vanished
  • No auth for the admin page, so anyone can request really big video download, and anyone can use cutout option.
